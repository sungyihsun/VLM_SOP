# Config File Reference

The config file is at `assets/config/augment_config.yaml`. Below is the full template:

```yaml
# General Config
video_extention: "mp4"    # Video file extension (note: typo is intentional, matches codebase)

# --- Default 4 Stages (enabled by default) ---

# BCQ Config
bcq:
  enable: true
  negative_ratio: 2.0       # 1 positive + N negative samples per video
  subject: "operator"        # Who performs the action
  exclude_action: ""         # Exclude actions by index: "1_2" = exclude action 1 and 2

# Sequential MCQ Config
sequential_mcq:
  enable: true
  exclude_action: ""
  max_chunk_len: 2           # Max actions per merged chunk (2 = single + pairs)

# Golden GQA Config
golden_gqa:
  enable: true

# GQA to GQAs Config (LLM-based expansion)
gqas:
  enable: true
  llm_type: "nvidia"                              # "nvidia" or "local"
  local_llm_url: ""                               # Required if llm_type is "local" (include /v1)
  llm: meta/llama-3.1-70b-instruct               # Model name
  num_qa_llm: 8                                   # QA pairs generated by LLM per action
  num_qa_per_chunk: 2                             # QA pairs sampled per video chunk
  exclude_action: ""
  # enable_thinking: ""                           # Optional: "true"/"false" for thinking-mode LLMs

# --- Advanced Stages (disabled by default) ---
# These require non_sop_action to be set.
# non_sop_action is the index of the "none of the above" / "doing action not belong to the defined SOP"
# action in your actions.json. This action MUST exist in your action list.

# Dynamic MCQ Config
dynamic_mcq:
  enable: false
  exclude_action: ""
  non_sop_action: 17         # MUST match your dataset's "none of the above" action index
  min_options: 3
  max_options: 6
  num_pos: 1                 # Positive samples per video
  num_neg: 2                 # Negative samples per video
  # num_hard_pos: 0          # Hard positive samples (requires hard_pos_mode)
  # num_hard_neg: 0          # Hard negative samples (requires hard_neg_mode)
  # hard_neg_mode: ""        # "adjacent", "confusion", or "adjacent,confusion"
  # hard_pos_mode: ""        # "adjacent", "confusion", or "adjacent,confusion"
  # confusion_map: ""        # JSON: "{action_idx: [confused_action_idx, ...]}"

# Dynamic Shuffling Config
dynamic_shuffling:
  enable: false
  exclude_action: ""
  non_sop_action: 17
  min_distractor: 3
  max_distractor: 6
  num_runs: 1                # Shuffled videos per original chunk

# Extra Negative Config
extra_negative:
  enable: false
  exclude_action: ""
  extra_negative_data_id: "<other_dataset_id>"    # MUST be a valid annotated dataset ID
  non_sop_action: 17
  min_options: 3
  max_options: 6
  num_runs: 1
  generate_all_options: true
```

## Key Concepts

**`non_sop_action`:** The action index for "none of the above" / "doing action not belong to the defined SOP." This must be a real action in your `actions.json`. It represents idle time, non-specific activity, or out-of-scope behavior. Required for dynamic_mcq, dynamic_shuffling, and extra_negative.

**`exclude_action`:** Underscore-separated action indices to skip during generation. Example: `"1_2"` excludes actions at index 1 and 2. Useful for excluding the non-SOP action from positive sample generation, or excluding trivially similar actions.

**`extra_negative_data_id`:** Must be the ID of a different annotated dataset already in `assets/data/`. This dataset's videos are used as negative examples for the current SOP.
